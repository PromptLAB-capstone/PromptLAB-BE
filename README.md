## 🚀 Git 컨벤션 규칙

### Commit 규칙

| 타입 이름 | 내용 |
| --- | --- |
| feat | 새로운 기능 추가 |
| fix | 버그 수정 |
| build | 빌드 및 모듈 설치/삭제 관련 변경 |
| chore | 기타 자잘한 수정 |
| ci | CI 설정 수정 |
| docs | 문서 업데이트 |
| style | 코드 스타일 및 포맷 수정 |
| refactor | 코드 리팩토링 |
| test | 테스트 코드 수정 |
| perf | 성능 개선 |

#### Commit Message Format
- **헤더(Header)**: `<타입>(스코프): <주제>`
- **본문(Body)**: 커밋의 상세 내용 (선택적)
- **바닥글(Footer)**: 관련 이슈 번호
